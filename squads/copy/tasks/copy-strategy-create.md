## Profile-Driven Approach

**CRITICAL:** This task reads from `content-manifest.md` and `brand-profile.yaml` for ALL decisions.
Do NOT hardcode deliverables, frameworks, tone, or channels. Everything is profile-driven.

### Step 0: Load Profile Context

1. Read `brand-profile.yaml` — extract:
   - `personality.archetypes` (primary + secondary)
   - `personality.tone_of_voice`
   - `personality.values`
   - `client.industry`
   - `client.target_audience`
2. Read `content-manifest.md` — extract:
   - Tone & Voice Profile (formality, energy, confidence, warmth, directness)
   - Content Pillars (names, weights, topics)
   - Content Types Selected (what was paid for)
   - Framework Assignments (which framework per content type)
   - Channel Strategy (selected channels)
3. Read `voice-guide.md` (if available) — extract vocabulary bank, forbidden words

## Workflow

### Phase 1: Messaging Architecture (15 min)

1. Define primary value proposition using brand archetype language:
   - **Use archetype signature phrases** from the content manifest
   - **Align with content pillars** — the value prop should touch the #1 pillar
   - **Speak to target audience** using language appropriate for their demographics
2. Create supporting messages (3-5 points) — one per content pillar
3. Establish proof points for each message (industry-appropriate)
4. Create objection handlers specific to the industry

### Phase 2: Hook Development (15 min)

1. Generate 10+ headline variants using archetype-appropriate angles:
   - Pain-focused (adjust intensity per archetype — Caregiver=gentle, Hero=bold)
   - Desire-focused (adjust aspiration per archetype — Explorer=adventure, Ruler=status)
   - Curiosity-driven (adjust mystery per archetype — Magician=secrets, Sage=insights)
   - Proof-based (adjust proof type per archetype — Ruler=data, Everyman=testimonials)
   - Contrarian (only for archetypes that support it: Outlaw, Jester, Explorer)
2. Score hooks by impact potential for the target audience

### Phase 3: Framework Assignment (10 min)

1. Read Framework Assignments from the content manifest
2. For each deliverable in the manifest, confirm or refine the assigned framework:

| Content Type | Framework Source | Notes |
|-------------|-----------------|-------|
| Landing page | content-manifest.md -> Framework Assignments | May have archetype override |
| Sales page | content-manifest.md -> Framework Assignments | Long-form vs short-form per scope |
| Email sequence | content-manifest.md -> Email Strategy | Framework per sequence type |
| Social post | content-manifest.md -> Framework Assignments | Platform-specific adjustments |
| Ad copy | content-manifest.md -> Ad Strategy | Channel-specific framework |

**Do NOT assign frameworks to content types that are NOT in the manifest.**

### Phase 4: CTA Strategy (10 min)

1. Define primary CTA per funnel stage, using brand tone:
   - Archetype tone shapes CTA language (Ruler="Request Access", Jester="Let's Go!")
   - Industry shapes CTA action (SaaS="Start Free Trial", Cafe="See Our Menu")
2. Create CTA variants for A/B testing
3. Define micro-CTAs for engagement on selected channels

## Acceptance Criteria

- [ ] Brand profile and content manifest loaded before strategy creation
- [ ] Value proposition uses archetype-appropriate language
- [ ] Messaging hierarchy maps to content pillars from manifest
- [ ] At least 10 hooks in hook bank with archetype fit documented
- [ ] Frameworks assigned match content manifest (not hardcoded defaults)
- [ ] CTAs use brand tone profile (formality, warmth levels)
- [ ] No deliverable addressed that is not in the content manifest
- [ ] Strategy approved by stakeholder

## Quality Gate
- Threshold: >70%
- Messaging hierarchy maps directly to content pillars from the manifest
- Hook bank contains at least 10 hooks with archetype fit documented per hook
- No deliverable addressed that is not present in the content manifest
