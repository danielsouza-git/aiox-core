# Copy Squad - Coding Standards

Standards for copy deliverables, formatting, and file structure.

## File Naming

### Copy Deliverables
```
{type}-{client/project}-{version}.md
```

Examples:
- `landing-page-acme-v1.md`
- `email-sequence-onboarding-v2.md`
- `ads-meta-summer-campaign-v1.md`

### Drafts and Revisions
```
{type}-{client/project}-draft-{number}.md
{type}-{client/project}-rev-{number}.md
```

Examples:
- `sales-page-product-x-draft-1.md`
- `sales-page-product-x-rev-2.md`

## Document Structure

### Standard Copy Document

```markdown
# [Copy Type]: [Title]

## Metadata
- **Client:** [Name]
- **Project:** [Project name]
- **Author:** [Agent/person]
- **Date:** [YYYY-MM-DD]
- **Version:** [X.X]
- **Status:** [Draft/Review/Approved/Final]

## Brief Summary
[1-2 sentences describing the deliverable]

## Target Audience
[Audience description]

## Copy Goal
[Primary objective]

---

## [COPY CONTENT]

---

## Notes
[Implementation notes, variations, etc.]

## Revision History
| Version | Date | Changes | Author |
|---------|------|---------|--------|
| 1.0 | YYYY-MM-DD | Initial draft | [Name] |
```

## Formatting Standards

### Headlines
- Use sentence case: "Get more leads today"
- NOT title case: "Get More Leads Today"
- Exception: Brand names, proper nouns

### Body Copy
- One idea per paragraph
- Max 3-4 lines per paragraph
- Use em dash (—) not double hyphens (--)
- Use smart quotes (" ") not straight quotes (" ")

### Lists
- Parallel structure (all start same way)
- Consistent punctuation (all periods or none)
- Max 7 items per list

### CTAs
- Use action verbs
- Specify the outcome
- Examples: "Get instant access", "Start your free trial", "Download the guide"

## Placeholders

Use brackets for variable content:

| Placeholder | Meaning |
|-------------|---------|
| `[First Name]` | Recipient's first name |
| `[Company]` | Company name |
| `[Product]` | Product name |
| `[Price]` | Price point |
| `[Deadline]` | Time-sensitive date |
| `[CTA Button]` | Call-to-action button text |
| `[Link]` | URL placeholder |

## Version Control

### Versioning System
- `v1.0` - Initial draft
- `v1.1` - Minor revisions
- `v2.0` - Major rewrite
- `v2.1` - Minor revisions to v2

### Status Labels
- `Draft` - Work in progress
- `Review` - Ready for review
- `Approved` - Client/stakeholder approved
- `Final` - Ready for implementation

## Quality Markers

Include at end of document:

```markdown
## Quality Checklist
- [ ] Spell-checked
- [ ] Grammar verified
- [ ] Brand voice aligned
- [ ] CTA optimized
- [ ] Mobile-friendly formatting
```

## Delivery Format

### For Web/Digital
- Markdown (.md) for drafts
- Google Docs for collaboration
- Copy to CMS for final

### For Print
- Include print specs
- Note any special characters
- Flag layout dependencies

### For Ads
- Include character counts
- Note platform requirements
- Provide all required variants
