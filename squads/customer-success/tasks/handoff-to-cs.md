# Handoff to CS

```yaml
task:
  id: handoff-to-cs
  name: "Handoff to CS"
  agent: onboarding-specialist
  squad: customer-success
  type: onboarding
```

## Proposito

Fazer a transicao formal do cliente do Onboarding para o time de CS/Retencao, garantindo que todo o contexto seja documentado e transferido. Apos o handoff, o Onboarding Specialist NAO continua acompanhando o cliente.

## Input

- Cliente com primeira vitoria alcancada (First Value completo)
- Historico completo de onboarding: boas-vindas, setup, first value, feedback
- Health score inicial do cliente

## Output

- Documento de handoff completo com todo o contexto do cliente
- CS/Retencao apresentado formalmente ao cliente
- Status no CRM atualizado para "Active - CS Ongoing"
- Onboarding encerrado

## Workflow

### Passo 1: Compilar documento de handoff
Reunir todo o contexto: dados do contrato, necessidades, configuracoes feitas, primeira vitoria, feedback, pontos de atencao.

### Passo 2: Definir responsavel CS
Junto com CS Head, definir quem sera o responsavel de CS para este cliente.

### Passo 3: Apresentar CS ao cliente
Fazer introdução formal entre cliente e responsavel de CS (via call ou email).

### Passo 4: Transferir contexto
Garantir que o responsavel de CS leu e entendeu o documento de handoff.

### Passo 5: Encerrar onboarding
Atualizar status no CRM e encerrar acompanhamento de onboarding.

## O que faz

- Compila documento de handoff com contexto completo
- Apresenta formalmente o responsavel de CS ao cliente
- Transfere todo o contexto para CS/Retencao
- Encerra o fluxo de onboarding

## O que NAO faz

- NAO continua acompanhando o cliente apos handoff
- NAO resolve problemas que surjam apos a transicao
- NAO define quem sera o responsavel de CS (CS Head decide)

## Ferramentas

- **Notion** -- Documento de handoff com template padrao
- **CRM** -- Atualizacao de status e responsavel
- **Email** -- Apresentacao formal ao responsavel CS
- **Zoom** -- Call de introdução se necessario
- **ClickUp** -- Registro de conclusao do onboarding

## Quality Gate

- Threshold: >70%
- Documento de handoff completo e entregue
- Responsavel de CS definido e informado
- Cliente apresentado ao responsavel de CS
- Status no CRM atualizado para "Active - CS Ongoing"
- Nenhuma informacao critica faltando no handoff
