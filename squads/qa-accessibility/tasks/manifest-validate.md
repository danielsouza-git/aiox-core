## Purpose

Validate the 4 manifests generated during Pipeline Phase 0b (Profile-Driven Configuration). Verify each manifest's internal structure, cross-validate references between manifests, and ensure no orphan references exist. This task runs BEFORE build phases (3, 4, 5) to catch configuration errors before code generation.

## Manifest Inventory

| Manifest | Path | Generated By | Consumed By |
|----------|------|-------------|------------|
| `component-manifest.md` | `{client_path}/component-manifest.md` | Phase 0b.2 (Component Selection Engine) | Phase 3 (Design System) |
| `motion-manifest.md` | `{client_path}/motion-manifest.md` | Phase 0b.3 (Animation Selection Engine) | Phase 4 (Visual) |
| `brandbook-manifest.md` | `{client_path}/brandbook-manifest.md` | Phase 0b.4 (Brand Book Structure Engine) | Phase 5 (Content) |
| `content-manifest.md` | `{client_path}/content-manifest.md` | Phase 0b (Content Strategy) | Phase 5 (Content) |

## Workflow

### Phase 1: Manifest Existence Check (2 min)

1. Verify each manifest file exists at expected path:
   - `{client_path}/component-manifest.md` -- REQUIRED
   - `{client_path}/motion-manifest.md` -- REQUIRED
   - `{client_path}/brandbook-manifest.md` -- REQUIRED
   - `{client_path}/content-manifest.md` -- OPTIONAL (warn if missing)
2. If any REQUIRED manifest is missing: FAIL immediately with clear guidance
3. Record file sizes and last-modified timestamps

### Phase 2: Component Manifest Validation (5 min)

Validate `component-manifest.md`:

1. **Profile Inputs section exists:**
   - Archetype is specified (non-empty)
   - Industry is specified (non-empty)
   - Scope is specified and valid (portfolio | small | medium | large | enterprise)

2. **Component selections present:**
   - At least 1 universal component listed
   - Each component has a selection reason (not empty)
   - Component count falls within scope range:

   | Scope | Min Components | Max Components |
   |-------|---------------|----------------|
   | Portfolio | 8 | 12 |
   | Small | 10 | 16 |
   | Medium | 12 | 22 |
   | Large/Enterprise | 16 | 30+ |

3. **Archetype styling presets defined:**
   - At least one styling preset specified
   - Presets reference valid design token categories (colors, typography, spacing)

4. **Exclusions documented:**
   - Components NOT included have documented reasons
   - No component is excluded without justification

5. **No arbitrary selections:**
   - Every included component has a reason tied to the brand profile
   - Flag any component with a vague reason (e.g., "just in case", "might be useful")

### Phase 3: Motion Manifest Validation (5 min)

Validate `motion-manifest.md`:

1. **Profile Inputs section exists:**
   - Archetype matches component-manifest archetype
   - Personality traits listed
   - Industry matches component-manifest industry
   - Scope matches component-manifest scope

2. **Animation Character defined:**
   - Motion personality specified
   - Default easing specified (valid CSS/Framer Motion easing)
   - Duration scale specified (numeric, typically 0.5-1.5)
   - Entrance style specified

3. **Motion component selections present:**
   - At least 2 motion components listed (fade-in and slide-up are universal)
   - Each component has a selection reason
   - `fade-in` is always present (universal requirement)

4. **Framer Motion Preset Overrides defined:**
   - Duration value specified
   - Stagger timing specified
   - Scale hover value specified (if applicable)
   - Spring stiffness and damping specified (if spring easing used)

5. **Exclusions documented:**
   - Motion components NOT included have documented reasons

### Phase 4: Brand Book Manifest Validation (5 min)

Validate `brandbook-manifest.md`:

1. **Profile Inputs section exists:**
   - Industry, archetype, scope, personality all specified
   - Values match other manifests

2. **Layout Decisions defined:**
   - Theme mode specified (dark-first | light-first | neutral) with reason
   - Layout style specified with reason
   - Navigation style specified with reason
   - None of the above are defaulted without justification

3. **Page Structure complete:**
   - Core pages listed (minimum 6)
   - Industry pages listed with selection reasons
   - Optional pages listed with justifications (if any)
   - Total page count falls within scope range:

   | Scope | Min Pages | Max Pages |
   |-------|-----------|-----------|
   | Portfolio | 10 | 12 |
   | Small | 12 | 16 |
   | Medium | 16 | 22 |
   | Large/Enterprise | 22 | 30+ |

4. **Routes defined:**
   - Every page has a route path specified
   - Routes follow Next.js App Router convention (`/brandbook/{section}/{page}`)
   - No duplicate routes

5. **Exclusions documented:**
   - Pages NOT included have documented reasons

### Phase 5: Content Manifest Validation (3 min)

If `content-manifest.md` exists, validate:

1. **Content types listed:**
   - Each content type has a corresponding template reference
   - Voice direction specified per content type
   - No empty required fields

2. **Alignment with brandbook manifest:**
   - Content types align with pages defined in brandbook-manifest
   - No content defined for pages that do not exist in the brandbook manifest

If `content-manifest.md` does not exist: WARN and continue.

### Phase 6: Cross-Manifest Validation (10 min)

Cross-validate references between all manifests:

1. **Profile Consistency:**
   - `archetype` is identical across all manifests
   - `industry` is identical across all manifests
   - `scope` is identical across all manifests
   - Flag any inconsistency as ERROR

2. **Component -> Brandbook alignment:**
   - Every component in component-manifest has at least one brandbook page where it could be showcased
   - Brandbook pages referencing "Component Catalog" or "Components" exist if components are defined
   - No brandbook page references a component category not in the component-manifest

3. **Motion -> Component alignment:**
   - If `hover-scale` is in motion-manifest, at least one interactive component exists in component-manifest
   - If `stagger-children` is in motion-manifest, a grid/list component exists
   - If `page-transition` is in motion-manifest, brandbook has multiple pages

4. **Motion -> Brandbook alignment:**
   - If brandbook-manifest includes "Motion Showcase" page, motion-manifest must have >= 4 motion components
   - If motion-manifest has `parallax-layer`, at least one brandbook page should showcase it

5. **Content -> Brandbook alignment (if content-manifest exists):**
   - Every page in brandbook-manifest has corresponding content defined
   - No content is defined for pages not in the brandbook-manifest

6. **Orphan Reference Detection:**
   - Scan all manifests for references to entities in other manifests
   - Verify each reference resolves to an actual entry
   - Report orphan references (references to non-existent entities)

### Phase 7: Completeness Check (3 min)

1. **No empty required fields:**
   - Scan all manifests for placeholder text (e.g., `{placeholder}`, `TBD`, `TODO`)
   - Flag any incomplete fields as ERROR

2. **No copy-paste artifacts:**
   - Check for generic template text that was not customized
   - Flag references to "reference site" or "template" as WARNING

3. **Justification quality:**
   - Every selection reason should be specific to the client
   - Flag generic reasons (e.g., "standard practice", "common choice") as WARNING

### Phase 8: Report Generation (2 min)

1. Compile all findings
2. Calculate overall manifest health score
3. Generate orphan references JSON

## Manifest Health Score

| Category | Weight | Pass Criteria |
|----------|--------|---------------|
| Existence | 15% | All required manifests present |
| Structure | 25% | All required sections and fields populated |
| Cross-Validation | 30% | Zero profile inconsistencies, zero orphan references |
| Completeness | 20% | No placeholders, no empty required fields |
| Justification Quality | 10% | All selections have client-specific reasons |

| Score | Rating | Verdict |
|-------|--------|---------|
| 100% | Perfect Manifests | Ready for build phases |
| 85-99% | Minor Issues | Proceed with documented warnings |
| 70-84% | Significant Issues | Fix before proceeding to build |
| < 70% | Manifest Failure | Regenerate manifests |

## Acceptance Criteria

- [ ] All required manifests verified as present
- [ ] Component manifest validated (selections, scope range, justifications)
- [ ] Motion manifest validated (animation character, presets, selections)
- [ ] Brandbook manifest validated (pages, layout decisions, routes)
- [ ] Content manifest validated if present
- [ ] Profile consistency verified across all manifests (archetype, industry, scope)
- [ ] Cross-references validated (no orphan references)
- [ ] No empty required fields or placeholder text
- [ ] Manifest validation report generated

## Quality Gate
- Threshold: >70%
- All required manifests present with complete structural fields
- Profile consistency verified across all manifests (archetype, industry, scope match)
- Zero orphan cross-references between manifests
