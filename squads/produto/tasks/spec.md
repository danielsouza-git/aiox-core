## Proposito

Documentar os requisitos do produto de forma clara para que o Content Creator saiba exatamente o que criar. A spec define o que o produto deve conter, criterios de aceitacao e restricoes. Para PRD formais e especificacoes detalhadas, delega para o agente AIOX core @pm (Morgan).

## Input

- Item priorizado do roadmap
- Discovery document da oportunidade
- Feedback de clientes relevante ao tema
- Restricoes tecnicas da plataforma

## Output

- Briefing de spec para Content Creator com requisitos claros
- Criterios de aceitacao definidos (o que e "pronto")
- PRD formal (quando necessario, via delegacao ao @pm Morgan)

## Workflow

### Passo 1: Definir escopo
Delimite o que o produto/conteudo deve cobrir e o que esta fora de escopo.

### Passo 2: Documentar requisitos
Escreva os requisitos de alto nivel: o que deve existir, para quem, em que formato.

### Passo 3: Definir criterios de aceitacao
Para cada requisito, defina o que significa "pronto" -- o que o CS/Retencao vai validar.

### Passo 4: Criar briefing para Content Creator
Transforme os requisitos em um briefing acionavel: o que criar, em que formato, para quando.

### Passo 5: Delegar PRD formal (se necessario)
Se o produto exige PRD formal, delegue para @pm (Morgan) com o contexto de negocio.

## O que faz

- Documenta requisitos de produto de alto nivel
- Define criterios de aceitacao claros
- Cria briefing acionavel para Content Creator
- Delega PRD formal para @pm (Morgan) quando necessario
- Define restricoes e dependencias

## O que NAO faz

- NAO escreve PRD formal sozinho (delega para @pm Morgan)
- NAO cria o conteudo (isso e do Content Creator)
- NAO define a arquitetura tecnica

## Ferramentas

- **Notion** -- Documentacao de specs e briefings
- **Figma** -- Prototipos e wireframes quando necessario
- **Slack** -- Alinhamento rapido com Content Creator
- **ClickUp** -- Rastreamento de tasks da spec

## Quality Gate

- Threshold: >70%
- Escopo claramente definido (o que entra e o que nao entra)
- Requisitos documentados e compreensivos
- Criterios de aceitacao definidos para cada requisito
- Briefing claro e acionavel para Content Creator
