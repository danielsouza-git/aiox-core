## Purpose

Create layout tokens in W3C DTCG format that define the structural and spatial personality of a brand. Layout tokens sit as a parallel tier alongside semantic tokens, addressing spatial composition (navigation, whitespace, corners, dividers, animations, grid, sections, components) rather than visual appearance.

## Workflow

```yaml
steps:
  - step: resolve_family
    description: Map brand archetypes to layout family using scoring algorithm
    input: [brand_profile.archetypes]
    output: layout_family (string)

  - step: load_family_preset
    description: Load base token values for the resolved family
    input: layout_family
    output: base_tokens (JSON)

  - step: modulate_by_personality
    description: Adjust token values based on personality trait scores (1-5)
    input: [base_tokens, personality_traits]
    rules:
      - formal 1 (casual) -> increase radiusBase, increase whitespace
      - formal 5 (formal) -> decrease radiusBase, tighten whitespace
      - bold 1 (subtle) -> lighter shadows, thinner dividers
      - bold 5 (bold) -> deeper shadows, thicker dividers
      - expressive 1 (minimal) -> fewer animations, simpler grid
      - expressive 5 (expressive) -> more animations, complex grid
    output: modulated_tokens (JSON)

  - step: validate_schema
    description: Validate output against W3C DTCG spec
    validator: w3c-dtcg-validator
    checks:
      - Every leaf has $value and $type
      - All string enums are valid values
      - All dimension values have units
      - No circular references

  - step: emit_layout_json
    description: Write layout.json to token output directory
    output: layout/layout.json
```
