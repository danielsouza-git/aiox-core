## Purpose

Transform W3C DTCG layout tokens (generated by Toren) into platform-specific outputs for BSS consumption. Layout tokens define structural/spatial personality properties and must be exported as:

1. **`styles/layout-tokens.css`** -- CSS custom properties with `--layout-*` prefix
2. **`lib/layout-tokens.ts`** -- TypeScript constants for programmatic access
3. **`tailwind.layout.ts`** -- Tailwind CSS 4 theme extension for layout properties

All outputs integrate with existing token outputs (tokens.css, tokens.ts, tailwind.config.ts) without conflict.

## CSS Custom Property Naming

Layout tokens use the `--layout-` prefix to avoid collision with existing tokens:

```css
:root {
  /* ===== Layout: Family ===== */
  --layout-family: ethereal;

  /* ===== Layout: Navigation ===== */
  --layout-nav-style: centered-top;
  --layout-nav-width: 100%;
  --layout-nav-height: 64px;

  /* ===== Layout: Whitespace ===== */
  --layout-whitespace-density: spacious;
  --layout-whitespace-multiplier: 1.5;
  --layout-whitespace-sectionGap: 96px;
  --layout-whitespace-contentPadding: 80px;

  /* ===== Layout: Corner ===== */
  --layout-corner-radiusBase: 24px;
  --layout-corner-radiusSmall: 16px;
  --layout-corner-radiusLarge: 32px;

  /* ===== Layout: Divider ===== */
  --layout-divider-style: organic-wave;
  --layout-divider-color: #e2e2ea;
  --layout-divider-height: 2px;

  /* ===== Layout: Animation ===== */
  --layout-animation-entrance: fade-up;
  --layout-animation-duration: 300ms;
  --layout-animation-easing: cubic-bezier(0.25, 0.1, 0.25, 1.0);

  /* ===== Layout: Grid ===== */
  --layout-grid-rhythm: centered-single;
  --layout-grid-maxWidth: 800px;
  --layout-grid-columns: 1;

  /* ===== Layout: Section ===== */
  --layout-section-background: soft-fill;
  --layout-section-heroHeight: 60vh;

  /* ===== Layout: Component ===== */
  --layout-component-cardShape: pill;
  --layout-component-shadowIntensity: light;
}
```

## Backward Compatibility

All CSS that references layout tokens MUST use the fallback pattern:

```css
border-radius: var(--layout-corner-radiusBase, 8px);
padding: var(--layout-whitespace-contentPadding, 48px);
```

If `--layout-*` properties are not defined (legacy builds without layout tokens), the CSS falls back to current hardcoded values (8px corners, standard padding, etc.).

## Workflow

### Phase 1: Validation

```yaml
steps:
  - validate_layout_schema: |
      Ensure layout token structure matches expected schema:
        - layout.family exists and is valid enum
        - All leaf nodes have $value and $type
        - All string enums contain allowed values only
        - All dimension values have valid CSS units
        - References to base tokens (e.g., {color.neutral.200}) resolve correctly

  - validate_family: |
      Verify layout.family.$value is one of:
        ethereal, bold-structured, warm-artisan,
        adventurous-open, playful-dynamic, rebel-edge
```

### Phase 2: Generate `styles/layout-tokens.css`

```yaml
steps:
  - flatten_tokens: |
      Flatten nested layout token structure to CSS custom property names:
        layout.nav.style -> --layout-nav-style
        layout.corner.radiusBase -> --layout-corner-radiusBase
        layout.whitespace.sectionGap -> --layout-whitespace-sectionGap

  - resolve_references: |
      Resolve token references to concrete values:
        {color.neutral.200} -> resolved hex value from base_tokens

  - emit_css: |
      Output all layout tokens as CSS custom properties under :root
      Organized by category with CSS comments
      String values unquoted (used as CSS identifiers or data attributes)
      Dimension values with units
      Number values as plain numbers
```

### Phase 3: Generate `lib/layout-tokens.ts`

```yaml
steps:
  - generate_constants: |
      Export TypeScript constants mirroring CSS custom properties:
        export const layoutTokens = {
          family: 'var(--layout-family)',
          nav: {
            style: 'var(--layout-nav-style)',
            width: 'var(--layout-nav-width)',
            height: 'var(--layout-nav-height)',
          },
          whitespace: { ... },
          corner: { ... },
          divider: { ... },
          animation: { ... },
          grid: { ... },
          section: { ... },
          component: { ... },
        } as const

  - generate_types: |
      Export union types for each layout enum:
        type LayoutFamily = 'ethereal' | 'bold-structured' | 'warm-artisan' | 'adventurous-open' | 'playful-dynamic' | 'rebel-edge'
        type NavStyle = 'centered-top' | 'sidebar-fixed' | 'breadcrumb-horizontal' | 'sticky-minimal' | 'floating-pill' | 'inline-minimal'
        type WhitespaceDensity = 'compact' | 'balanced' | 'generous' | 'spacious'
        type DividerStyle = 'solid-thin' | 'solid-thick' | 'organic-wave' | 'textured-line' | 'thin-geometric' | 'zigzag-wave' | 'slash-raw' | 'none'
        type GridRhythm = 'centered-single' | 'strict-grid' | 'masonry-inspired' | 'editorial-wide' | 'broken-asymmetric' | 'single-column-stacked'
        type CardShape = 'sharp' | 'subtle' | 'rounded' | 'pill'
```

### Phase 4: Generate `tailwind.layout.ts`

```yaml
steps:
  - generate_extension: |
      Create Tailwind theme extension for layout properties:
        - borderRadius: map layout corner tokens
        - gap/padding: map layout whitespace tokens
        - maxWidth: map layout grid tokens
        - transitionDuration: map layout animation tokens
        - Custom utilities for layout family classes
```

### Phase 5: Generate Transform Report

```yaml
steps:
  - count_tokens: |
      Count layout tokens per category:
        - Total layout tokens
        - Navigation (3), Whitespace (4), Corner (3)
        - Divider (3), Animation (3), Grid (3)
        - Section (2), Component (2)
        - Family (1)
  - verify_completeness: |
      Confirm all layout tokens appear in all three outputs
  - document_family: |
      Include resolved family name and key characteristics
```
