# Swipe File

```yaml
task:
  id: swipe-file
  name: "Swipe File"
  agent: research-analyst
  squad: marketing
  type: research
```

## Proposito

Manter um acervo organizado de criativos vencedores (anuncios, emails, landing pages, posts) categorizados por tipo, plataforma e formato, servindo como referencia permanente para COPY e Media Buyer na criacao de novos materiais.

## Input

- Criativos de alto desempenho identificados durante Competitor Analysis e Trend Hunting
- Anuncios vencedores do Media Buyer (internos)
- Referencias encontradas em plataformas e bibliotecas de anuncios
- Criativos destacados pela equipe no dia a dia

## Output

- Swipe file organizado por categoria (video, carrossel, copy, email, landing page)
- Tags por plataforma, formato, setor e objetivo
- Referencias prontas para COPY e Media Buyer consultarem
- Acervo atualizado semanalmente

## Workflow

### Passo 1: Coletar criativos vencedores
Salvar criativos de alto desempenho encontrados durante analise competitiva, trend hunting e monitoramento diario.

### Passo 2: Categorizar
Organizar cada criativo por: tipo (video, carrossel, copy, email, LP), plataforma (Instagram, TikTok, Facebook, Google), formato (stories, feed, search), setor e objetivo (awareness, conversao, retargeting).

### Passo 3: Adicionar contexto
Para cada criativo, documentar: por que funciona, elementos que se destacam, hook utilizado, CTA, estimativa de performance.

### Passo 4: Atualizar acervo
Adicionar novos criativos semanalmente e remover os desatualizados ou irrelevantes.

### Passo 5: Disponibilizar para o squad
Garantir que COPY e Media Buyer tenham acesso facil ao swipe file como referencia para criacao.

## O que faz

- Salva criativos vencedores de concorrentes e do mercado
- Organiza por categoria, plataforma, formato e objetivo
- Documenta por que cada criativo funciona
- Alimenta COPY e Media Buyer com referencias de qualidade
- Mantem acervo atualizado semanalmente
- Remove criativos desatualizados

## O que NAO faz

- NAO copia criativos diretamente (usa APENAS como referencia)
- NAO cria conteudo baseado nos criativos (pede a COPY)
- NAO roda anuncios (isso e do Media Buyer)
- NAO decide quais criativos replicar (recomenda, COPY adapta)

## Ferramentas

- **Notion** -- Organizacao e documentacao do swipe file
- **TikTok** -- Fonte de criativos de video
- **Twitter** -- Fonte de criativos de copy e threads
- **SimilarWeb** -- Referencia de landing pages de concorrentes

## Quality Gate

- [ ] Acervo atualizado semanalmente com novos criativos
- [ ] Criativos categorizados com tags corretas
- [ ] Contexto documentado (por que funciona) para cada criativo
- [ ] COPY e Media Buyer com acesso facil ao acervo
- [ ] Criativos desatualizados removidos mensalmente
